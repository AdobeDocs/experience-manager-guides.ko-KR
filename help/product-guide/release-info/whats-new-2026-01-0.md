# 2026.01.0 릴리스의 새로운 기능(2026년 2월)

이 문서에서는 Adobe Experience Manager Guides as a Cloud Service 2026.01.0 릴리스와 함께 도입된 새로운 기능 및 향상된 기능을 다룹니다.

이 릴리스에서 해결된 문제 목록을 보려면 [2026.01.0 릴리스에서 해결된 문제](fixed-issues-2026-01-0.md)를 확인하십시오.

[2026.01.0 릴리스의 업그레이드 지침](../release-info/upgrade-instructions-2026-01-0.md)에 대해 알아봅니다.


## 찾기 및 바꾸기의 Source 모드 검색 소개

Experience Manager Guides은 편집기 인터페이스의 왼쪽 패널에서 사용할 수 있는 찾기 및 바꾸기 기능에 대한 몇 가지 개선 사항을 도입했습니다. 유용성 향상을 위한 개선된 UI와 함께 이번 릴리스에서는 **찾기 및 바꾸기** 패널에서 새로운 **소스 모드 사용** 토글을 도입했습니다.

이 모드를 활성화하면 표시된 컨텐츠뿐만 아니라 검색된 문자열의 기본 소스 컨텐츠(요소, 태그 및 속성 값을 포함한 XML 구조)에 대해서도 글로벌 검색을 수행할 수 있습니다. 이 모드에서는 전체 콘텐츠를 포괄적으로 검색할 수 있습니다.

![](assets/map-find-replace-with-source-mode.png){width="650" align="left"}

이 모드에서 필터를 적용하여 파일 유형, 문서 상태, 마지막으로 수정한 날짜 등을 기준으로 검색 범위를 좁힐 수 있습니다. 또한 모두 바꾸기 작업을 수행한 후 자세한 CSV 보고서를 다운로드할 수 있는 옵션이 있으며, 이 옵션은 성공 및 실패 상태와 함께 수행된 모든 바꾸기 작업에 대한 스냅샷을 제공합니다.

자세한 내용은 편집기의 [왼쪽 패널](../user-guide/web-editor-left-panel.md#find-and-replace)에서 _찾기 및 바꾸기_ 섹션을 참조하십시오.

>[!NOTE]
>
> [찾기 및 바꾸기] 패널의 **원본 모드 사용** 기능의 경우 먼저 사용자 지정 인덱스 배포를 완료해야 합니다. 색인화가 실행되면 고객 지원 팀에 문의하여 이 기능을 활성화하십시오.

## 파일 및 폴더 검색 환경 개선

이 릴리스에서는 Experience Manager Guides의 파일 및 폴더 경로를 검색할 수 있는 보다 깔끔하고 직관적인 인터페이스를 도입했습니다.

파일을 탐색할 때 개정된 **파일 선택** 대화 상자에는 이제 두 가지 보기 옵션이 있는 탭 레이아웃이 제공됩니다. **저장소**&#x200B;는 전체 콘텐츠 리포지토리를 표 형식으로 탐색하고, **컬렉션**&#x200B;은(는) 자주 사용하는 항목, 맵 및 이미지에 빠르게 액세스할 수 있습니다.

![](assets/select-file.png){width="650" align="left"}

주요 개선 사항은 다음과 같습니다.

- 구성 탐색을 위한 파일 및 폴더의 표 형식 보기입니다.
- 이동 경로 및 폴더 탐색 패널을 통해 폴더를 쉽게 이동할 수 있습니다.
- 재사용 가능한 콘텐츠, 주제 참조, 스키마, 출력 사전 설정(DITAVAL 사용) 및 Workfront에 대한 다중 파일 선택을 지원합니다.
- 쉽게 검토할 수 있도록 선택한 파일을 미리 봅니다. 여러 항목을 선택하려면 모든 파일을 미리 보고 필요에 따라 [미리 보기] 패널에서 아무 항목이나 제거합니다.
- 검색 및 필터링 옵션을 통해 이름, 제목, 파일 유형, 문서 상태 및 태그로 검색 결과의 범위를 좁힐 수 있습니다.

**경로 선택** 대화 상자에는 폴더 탐색을 위한 개선된 트리 구조 보기가 포함되어 있어 콘텐츠 저장소에서 경로를 보다 체계적이고 효율적으로 선택할 수 있습니다.

![](assets/select-path-dialog-new.png){width="350" align="left"}

자세한 내용은 [편집기의 기타 기능](../user-guide/web-editor-other-features.md#browse-files-and-folders-in-experience-manager-guides)에서 _Experience Manager Guides에서 파일 및 폴더 찾아보기_ 섹션을 참조하십시오.

## 저장소 검색 및 필터 개선 사항

### 문서 상태 필터 지원

이제 파일의 현재 문서 상태를 기반으로 저장소 검색 결과를 필터링합니다. 새 **문서 상태** 필터를 사용하면 폴더 프로필 내의 `ui_config.json` 파일에 정의된 사용 가능한 필터 값을 사용하여 검색 범위를 좁힐 수 있습니다.

![](assets/document-state-filter-repository.png){align="left"}

문서 상태에 사용할 수 있는 기본 필터 값은 초안, 편집, 검토 중, 승인됨, 검토됨 및 완료됨입니다. 기본 문서 상태 필터 값 사용자 지정에 대한 자세한 내용은 [문서 상태 필터 구성](../cs-install-guide/config-doc-state-filters.md)을 참조하세요.

>[!NOTE]
>
> `ui_config.json`에 대한 사용자 지정 설정을 사용하는 경우 업그레이드하기 전에 이러한 설정을 백업해야 합니다. 업데이트 후 설정을 검토하고 조정하여 최신 버전에 도입된 변경 내용에 맞게 조정합니다.

### 멀티미디어용 썸네일 아이콘

이제 모든 멀티미디어 파일이 썸네일 아이콘과 함께 표시되므로 **저장소**&#x200B;에서 이미지를 보다 쉽게 시각적으로 식별하고 찾을 수 있습니다. 이 개선 사항은 **검색 패널**&#x200B;에서 파일을 검색할 때도 적용되므로 멀티미디어 에셋을 다른 파일 유형과 빠르게 구별할 수 있습니다.

![](assets/thumbnail-repository.png){align="left"}

## 편집기 개선 사항

이 릴리스의 일부로 다음과 같은 편집기 개선이 이루어졌습니다.

### 미리보기 모드에서 주제 새로 고침 또는 매핑

미리 보기 모드에서 이미 열려 있는 맵에 대해 새로운 **새로 고침** 기능을 도입하고 있습니다. 이 새로운 기능을 사용하면 전체 맵의 콘텐츠나 맵 내에 있는 개별 주제를 쉽게 새로 고칠 수 있습니다.

- 전체 맵(모든 항목 포함)을 새로 고치기 위해 편집기의 왼쪽 상단 모서리에 새 **새로 고침** 단추가 도입되었습니다.

  ![](assets/refresh-map.png){width="600" align="left"}

- 개별 주제의 내용을 새로 고치려면 컨텍스트 메뉴에 새로운 **주제 새로 고침** 옵션이 도입되었습니다.

  ![](assets/refresh-topic.png){width="600" align="left"}

자세한 내용은 [맵 편집기 기능](../user-guide/map-editor-advanced-map-editor.md)을 참조하세요.

### 메타데이터 변경에 대한 작업 복사 표시기

**파일 속성**&#x200B;에서 사용할 수 있는 메타데이터 필드를 변경하면 이제 작업 복사 표시기가 트리거됩니다. 기본 또는 사용자 지정 메타데이터 필드를 추가, 삭제 또는 수정할 때마다 문서 버전이 _더티(*)_(으)로 표시됩니다. 이 향상된 기능을 통해 모든 메타데이터 변경 사항이 정확하게 추적되므로 문서 버전에 대한 가시성과 제어 능력이 향상됩니다.

### 주제 및 맵의 단어 수

이제 맵 또는 주제 파일 내에 있는 단어 수를 추적할 수 있습니다. 오른쪽 패널의 새 **단어 수** 필드에는 주제(또는 맵) 내에 있는 총 단어 수가 표시됩니다. 여기서 공백으로 구분된 단어는 개별 단어로 계산됩니다. 변경 사항을 저장할 때마다 자동으로 새로 고쳐집니다. 상호 참조의 경우 표시 텍스트만 포함되고 키는 제외됩니다.

![](assets/file-properties-new.png){width="350" align="left"}

자세한 내용은 편집기의 [오른쪽 패널](../user-guide/web-editor-right-panel.md#file-properties)을 참조하세요.

### 읽기 전용 파일에 대한 처리가 개선되었습니다.

이제 **읽기 전용** 모드에 있는 파일에 대한 파일 속성 편집이 제한됩니다. 다른 사용자가 파일을 잠근 경우(읽기 전용 모드에서 사용 가능), [오른쪽 패널](../user-guide/web-editor-right-panel.md#file-properties), **파일의 컨텍스트 메뉴**&#x200B;에 있는 [속성](../user-guide/web-editor-other-features.md#context-menu-functions-on-a-files-tab) 옵션 또는 [메타데이터 보고서](../user-guide/reports-web-editor.md#metadata-report)에서 메타데이터 속성을 변경할 수 없습니다. 이렇게 하면 읽기 전용 파일에 대한 우발적 변경을 방지할 수 있습니다.

## 향상된 기능 검토

### 진행 중인 검토 작업에서 주제 추가 또는 제거

이제 검토 워크플로에 영향을 주지 않고 진행 중인 검토 작업에 새 주제를 추가하거나(이전에 검토 대상으로 전송되지 않은 경우) 진행 중인 검토 작업에서 주제를 제거할 수 있습니다.

**작업 세부 정보** 페이지에서 주제를 선택하거나 선택 취소하여 주제 목록을 수정할 수 있습니다. 검토자는 AEM 및 이메일을 통해 AEM 및 이메일 알림을 통해 할당된 주제의 변경 사항에 대한 알림을 받습니다. 자세한 내용은 [검토할 항목 보내기](../user-guide/review-send-topics-for-review.md)를 참조하십시오.

![](assets/modify-review-topics.png){width="650" align="left"}

## 번역 개선 사항

### 번역을 위해 전송된 버전 없는 에셋에 대한 표시기

번역을 관리할 때는 처리를 위해 콘텐츠를 보내기 전에 모든 콘텐츠의 버전을 지정해야 합니다. 이를 위해 Experience Manager Guides에서는 이제 변경 사항을 저장했지만 아직 버전이 관리되지 않은 주제에 대한 명확한 표시기를 제공합니다.

파일에 버전이 없는 변경 내용이 포함되어 있으면(맵에 새 버전으로 저장되지 않음) 파일 옆에 업데이트가 있음을 나타내는 _info_ 아이콘이 나타납니다. 이러한 파일에 빠르게 집중하려면 [필터] 패널에서 **버전 없는 변경 내용만 포함된 자산 표시** 옵션을 활성화하십시오.

자세한 내용은 [맵 콘솔에서 문서 번역](../user-guide/translate-documents-web-editor.md)을 참조하세요.

![](assets/unversioned-changes-translation.png){width="650" align="left"}

## 향상된 게시 기능

### 특정 출력 사전 설정에 대한 사용자 지정 이미지 렌디션

이제 `outputName`의 `renditionmapping.xml` 특성을 사용하여 동일한 출력 유형에서 개별 출력 사전 설정에 대해 서로 다른 이미지 변환을 구성할 수 있습니다. 이러한 향상된 기능을 통해 다양한 시나리오에 대해 다양한 이미지 해상도가 필요한 콘텐츠를 게시할 때 보다 유연하게 작업할 수 있습니다. 예를 들어 가벼운 사전 설정에 대해 더 작은 썸네일을 사용하는 동안 주 HTML5 출력에 대한 고해상도 이미지를 원할 수 있습니다.

자세한 내용은 [출력 생성에서 이미지 렌디션 처리](../cs-install-guide/conf-output-generation.md#handle-image-rendition-during-output-generation)를 참조하십시오.


### 생성된 출력에 대한 로그 다운로드

이제 Assets UI를 통해 출력을 생성할 때 보다 쉽게 액세스하고 검토할 수 있도록 로컬 장치에 로그를 다운로드할 수 있는 새로운 **로그 다운로드** 단추를 사용할 수 있습니다.


### 기본 PDF 출력의 상호 참조에 대한 언어 변수

네이티브 PDF 출력을 게시할 때 [언어 변수](../native-pdf/native-pdf-language-variables.md)를 사용하여 _챕터 참조_ 또는 _페이지에서 참조_&#x200B;와 같은 정적 상호 참조 텍스트를 번역할 수 있습니다. 변수는 `xml:lang` 특성을 통해 주제에 정의된 언어를 사용합니다.

기본 PDF 출력 사전 설정 및 상호 참조 설정을 구성하는 방법에 대한 자세한 내용은 [기본 PDF 출력 사전 설정](../web-editor/native-pdf-web-editor.md)을 참조하세요.


### 새 AEM Sites(복합 구성 요소 매핑 사용) 게시에서 요소 수준 구성 요소 매핑 지원

Experience Manager Guides은 이제 AEM Sites 출력에서 요소 수준 구성 요소 매핑을 지원하므로(복합 구성 요소 매핑 사용) 팀에게 `componentmapping.json`을(를) 사용하여 DITA 요소가 렌더링되는 방법을 정확하게 제어할 수 있습니다. `topicref`, 제목, 이미지, 표 등을 적절한 AEM 핵심 구성 요소에 매핑하면 텍스트 구성 요소에 기본값으로 설정되는 모든 것 대신 더 깔끔한 구조를 얻을 수 있습니다. 이렇게 하면 성능이 향상되고 보다 풍부하고 현대적인 Sites 환경이 잠금 해제됩니다.

자세한 내용은 [AEM Sites의 구성 요소 매핑](../cs-install-guide/component-mapping.md)을 참조하세요.

## 자산 처리 개선 사항

이번 릴리스에서는 자산 처리에 대해 다음과 같은 개선 사항이 도입되었습니다.

- 폴더 및 개별 파일 수준 모두에서 자산 처리 실행
- 주제, 맵, Markdown, HTML/CSS, DITAVAL 또는 기타 지원되는 파일과 같은 특정 자산 유형을 선택하여 자산을 필터링하여 필요한 파일만 처리합니다.
- 날짜 기반 필터를 적용하여 지정된 일정의 처리 범위를 제한합니다.
- 저장소 보기 및 탐색기 패널 내의 파일 및 폴더의 컨텍스트 메뉴에서 사용할 수 있는 새 옵션(**에셋 재처리**)을 사용하여 에셋을 직접 재처리합니다.

자산 처리에 대한 자세한 내용은 [자산 처리](../user-guide/asset-processor.md)를 참조하십시오.

## API 개선 사항

이 릴리스의 일부로 다음과 같은 API 개선이 이루어졌습니다.

- 새 API는 새 번역 프로젝트를 만들고 상태를 추적하기 위해 도입되었습니다. 이러한 API는 번역 프로세스를 자동화하여 수작업을 줄이고 효율성을 개선하는 데 도움이 됩니다. 자세한 내용은 [번역 프로젝트 만들기](../api-reference/translation-project.md)를 참조하세요.
- 파일 및 폴더에 대한 향상된 필터링 기능으로 자산 처리 API를 개선했습니다. 자세한 내용은 [자산 처리](../api-reference/bulk-assets-processing.md)를 참조하세요.
