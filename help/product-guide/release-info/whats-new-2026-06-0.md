# 2026.06.0 릴리스(2026년 6월)의 새로운 기능

이 문서에서는 Adobe Experience Manager Guides as a Cloud Service 2026.06.0 릴리스와 함께 도입된 새로운 기능 및 향상된 기능을 다룹니다.

이 릴리스에서 수정된 문제 목록을 보려면 [2026.06.0 릴리스에서 수정된 문제](fixed-issues-2026-06-0.md)를 확인하십시오.

[2026.06.0 릴리스의 업그레이드 지침](../release-info/upgrade-instructions-2026-06-0.md)에 대해 알아봅니다.


## 맵 관리 및 출력 게시용 새 맵 컬렉션(Beta)

>[!NOTE]
>
> 이 기능은 현재 Beta 기능으로 사용할 수 있으며 기본적으로 비활성화되어 있습니다. 환경에서 활성화하려면 고객 지원 팀에 문의하십시오.

새로운 맵 컬렉션(Beta)은 맵 컬렉션 관리 및 출력 생성 활동을 단일 인터페이스로 통합합니다. 한 위치에서 맵 및 사전 설정을 관리하고, 출력을 생성하고 게시하며, 생성 및 게시 기록을 보는 등의 작업을 수행할 수 있습니다. 관련 게시 작업을 함께 가져오면 맵 컬렉션을 더 쉽게 사용하여 작업하고 여러 맵과 관련 언어에서 출력 활동을 추적할 수 있습니다.

![](assets/new-maps-collection.png)

자세한 내용은 [출력 생성을 위해 새 맵 컬렉션 사용(Beta)](../user-guide/generate-output-use-new-map-collection-output-generation.md)을 참조하세요.

## 기본 PDF용 새 게시 엔진 소개

이제 Experience Manager Guides의 기본 PDF에 새 게시 엔진인 *기본 PDF 엔진 v2*&#x200B;을(를) 사용할 수 있습니다. 여기에는 렌더링 개선 사항 및 기본 PDF 엔진 v1 문제에 대한 수정 사항이 포함되어 있습니다. 렌더링 동작이 업데이트되었으므로 *기본 PDF 엔진 v2*&#x200B;을(를) 사용하여 생성된 PDF 출력은 기존 기본 PDF 게시 엔진 *기본 PDF 엔진 v1*&#x200B;을(를) 사용하여 생성된 출력과 다를 수 있습니다.

예를 들어 *기본 PDF 엔진 v2*&#x200B;에서 사용하는 핵심 글꼴의 업데이트로 인해 생성된 PDF의 텍스트가 약간 다르게 나타날 수 있습니다. 마찬가지로 이미지 보간 및 렌더링 동작의 개선으로 인해 이미지가 더 선명하게 나타날 수 있습니다.

사용자 환경에서 [기본 PDF 엔진 v2](../native-pdf/conf-new-pdf-engine.md)을(를) 사용하는 방법에 대해 알아봅니다.

**기본 PDF 엔진 v2**&#x200B;에 대한 자세한 내용은 [기본 PDF 엔진 v2](../native-pdf/new-pdf-engine.md)에서 작업 을 참조하십시오.


## 편집기 개선 사항

### AMA 인용 스타일 지원

Experience Manager Guides은 이제 미국 의료 협회(AMA) 인용 스타일을 지원하여 의료, 규제 및 생명 과학 분야에서 고객이 요구하는 문서 표준을 충족하도록 기존 인용 프레임워크를 확장합니다.

**Workspace 설정**&#x200B;에서 AMA를 인용 스타일로 선택하면 숫자 위 첨자 렌더링, 일련 번호 매기기 및 정확한 참조 목록 순서 지정 등 AMA 지침에 따라 인용 형식이 자동으로 지정됩니다. 편집기의 **구문 분석 인용** 옵션은 AMA를 선택한 경우에만 사용할 수 있으므로 작성자는 컨텍스트를 전환하지 않고 인용을 추가하고 구문 분석할 수 있습니다.

AMA 인용 스타일은 기본 PDF 및 AEM Sites 출력 형식 전반에서 지원됩니다. 인용 스타일을 구성하려면 **Workspace 설정**(으)로 이동하여 인용 스타일 옵션에서 AMA를 선택하십시오. 자세한 내용은 [피인용 작업](../user-guide/web-editor-apply-citations.md)을 참조하세요.


### 이제 새 편집기에서 외부 데이터 소스 및 인용에 대한 지원을 사용할 수 있습니다

새 편집기는 이제 외부 데이터 소스와 연결하고 문서의 인용 부호를 사용하는 기능이라는 두 가지 기존 Experience Manager Guides 기능을 지원합니다.

작성자는 새 편집기에서 콘텐츠를 만들거나 업데이트하는 동안 구성된 외부 데이터 소스를 계속 사용할 수 있습니다. 또한 인용이 지원되므로 작성자는 편집기를 전환하지 않고도 콘텐츠에 참조를 추가하고 관리할 수 있습니다.

## 향상된 기능 검토

### 검토 UI와 AEM 받은 편지함(Beta) 간 검토 작업 완료 동기화

>[!NOTE]
>
> 이 기능은 현재 Beta 기능으로 사용할 수 있으며 기본적으로 비활성화되어 있습니다. 환경에서 활성화하려면 고객 지원 팀에 문의하십시오.

이제 검토 UI와 AEM 받은 편지함 간에 검토 작업 완료를 계속 동기화할 수 있습니다. 이 기능이 활성화되면 검토 UI에서 작업을 완료하면 AEM 받은 편지함에서 작업이 제거되고 AEM 받은 편지함에서 작업을 완료하면 검토 UI에서 완료된 것으로 표시됩니다. 이렇게 하면 동일한 작업을 두 번 완료하지 않고 검토 워크플로를 더 원활하게 수행할 수 있습니다. 작성자 및 작업 개시자는 피드백을 계속 검토하고 추가 검토가 필요한 경우 작업을 재할당할 수 있습니다. 작업이 재할당되면 검토자에 대해 새로운 AEM 받은 편지함 알림이 생성되므로 검토 주기가 원활하게 계속될 수 있습니다.

자세한 내용은 [검토자로 검토 작업을 완료](../user-guide/review-complete-review-tasks.md)하세요.

## 학습 콘텐츠 개선 사항

이 릴리스의 제품 교육 및 학습 콘텐츠 기능에는 다음과 같은 개선 사항이 있습니다.

- 이제 작성자는 학습자가 과정을 진행하기 전에 지식 확인을 필수 항목으로 지정할 수 있습니다. 과정의 지식 확인을 위해 새 **지식 확인 필요** 옵션이 도입되었습니다. 활성화되면 학습자는 후속 강의 콘텐츠로 진행하기 전에 지식 확인을 시도해야 합니다. 이렇게 하면 과정의 지정된 지점에서 필요한 지식 검사가 완료될 수 있습니다. 자세한 내용은 삽입 메뉴의 [기타 옵션](../learning-content/lc-other-insert-options.md)을 참조하세요.
- 이제 학습 콘텐츠를 만들 때 여러 줄 텍스트 입력 필드를 사용할 수 있습니다. 이러한 향상된 기능을 통해 사용자 지정 스크립팅에 의존하지 않고 단일 필드 내에서 줄 바꿈과 텍스트 줄바꿈을 지원하여 더 긴 학습자 응답을 더 쉽게 캡처할 수 있습니다. [삽입 메뉴의 기타 옵션](../learning-content/lc-other-insert-options.md)에 대해 자세히 알아보세요.
- 이제 SCORM 출력 템플릿에서 과정 내의 다른 주제 유형에 다른 페이지 레이아웃을 할당할 수 있습니다. 이렇게 하면 출력 템플릿 설정에서 직접 레슨, 퀴즈, 개요 페이지 및 기타 주제 유형에 대한 전용 레이아웃을 구성할 수 있습니다.

  이렇게 하면 각 주제 유형이 모든 강의 페이지에 동일한 레이아웃을 적용하지 않고 내용 및 구조에 적합한 레이아웃을 사용할 수 있습니다. SCORM 출력 템플릿의 페이지 레이아웃을 구성하는 방법에 대한 자세한 내용은 [폴더 프로필 구성](../lc-config-guide/lc-folder-profile.md)을 참조하십시오.
- Experience Manager Guides은 이제 SCORM 콘텐츠를 ALM(Adobe Learning Manager)에 직접 게시할 수 있도록 지원합니다. 작성자는 ALM 게시 프로필을 구성한 후 SCORM 출력을 생성하고 패키지를 다운로드하여 수동으로 가져오지 않고 Adobe Learning Manager에 직접 업로드할 수 있습니다.

  자세한 내용은 [SCORM 사전 설정 구성](../learning-content/config-scorm-preset.md)을 참조하십시오.
