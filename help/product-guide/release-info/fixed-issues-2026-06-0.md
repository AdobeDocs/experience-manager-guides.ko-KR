# 2026.06.0 릴리스의 문제가 해결되었습니다.

이 문서에서는 Adobe Experience Manager Guides as a Cloud Service의 2026.06.0 릴리스의 다양한 영역에서 수정된 버그를 다룹니다.

새로운 기능 및 향상된 기능에 대한 자세한 내용은 [2026.06.0 릴리스의 새로운 기능](whats-new-2026-06-0.md)을 참조하세요.

[2026.06.0 릴리스의 업그레이드 지침](upgrade-instructions-2026-06-0.md)에 대해 알아봅니다.

## 작성

- `in`, `mm` 또는 `px`과(와) 같은 단위 기반 크기를 사용하여 이미지 속성 대화 상자의 **Width** 및 **Height** 필드 간에 포커스를 전환하는 경우 값이 안정적으로 유지되지 않고 증분 방식으로 계속 증가합니다. (GUIDES-45929)

## 편집기 2.0

- `<keydef>` 또는 `<topicref>` 내에서 `<topicmeta>` 내의 `<keywords>`을(를) 복사하여 붙여넣으면 키워드가 원치 않는 외부 태그 내에 삽입됩니다. (GUIDES-45800)
- `mm`과(와) 같은 단위로 지정된 이미지 차원이 올바르게 렌더링되지 않아 이미지가 지정된 크기 대신 원래 크기로 표시됩니다. (GUIDES-46275)
- 태그 보기를 활성화한 상태로 드래그 앤 드롭을 수행할 때 부분 XML 또는 DITA 태그와 함께 컨텐츠를 선택하면 원하지 않는 고립된 태그가 뒤쳐져 잘못된 컨텐츠 또는 보기가 생성됩니다. (GUIDES-28191)
- 테이블의 태그 보기에서, 커서가 축소된 항목 태그 바로 아래 셀에 있을 때 위쪽 화살표 키를 누르면 축소된 태그가 건너뛰고 커서가 문서의 시작으로 이동합니다. (GUIDES-45408)
- 테이블 상황별 도구 모음에서 작업을 수행하면 예기치 않게 도구 모음이 닫히므로 후속 테이블 작업이 중단됩니다. (GUIDES-45405)
- 개요 보기 또는 탐색 표시에서 **다음 항목 뒤에 삽입** 또는 **다음 항목 앞에 삽입**&#x200B;을 사용한 후 커서가 새로 추가된 태그 내부가 아닌 임의의 위치로 이동합니다. (GUIDES-45147)
- **MathML 편집** 옵션이 읽기 전용 모드로 잘못 표시되거나 다른 사용자가 파일을 체크 아웃한 경우 파일을 편집할 수 없더라도 사용자가 MathML 콘텐츠를 업데이트할 수 있습니다. (GUIDES-45172)
- 백스페이스 키를 사용하여 XML 주석을 삭제할 때 주석 태그는 해당 콘텐츠가 삭제된 후에도 제거되지 않으며 삭제 작업이 이전 요소로 계속됩니다. (GUIDES-45240)
- 이미지에 대해 **경로 복사** 또는 **UUID 복사** 옵션을 사용하여 원본 에셋 경로 대신 전체 렌디션 경로를 반환합니다. (GUIDES-45278)
- 동일한 주제의 내용을 편집기의 잘못된 위치에 복사하여 붙여넣으면 의도하지 않은 `<foreign>` 태그가 삽입됩니다. (GUIDES-45378)
- Windows에서 표 행을 복사하여 붙여 넣으면 예기치 않은 특성(예: `data-pm-slice`)이 표 요소에 추가되어 마크업 오류가 발생하여 문서가 저장되지 않습니다. (GUIDES-45784)
- 맵 또는 항목의 컨텍스트 메뉴에서 **복사** 옵션을 사용한 다음 콘텐츠를 붙여넣으면 붙여넣은 출력에 `<data>`개의 태그가 추가로 삽입됩니다. (GUIDES-45703)
- `cmd` 요소에서 일부 선택 항목을 드래그할 때 기존 텍스트 사이 또는 다른 `cmd` 요소의 끝에서 컨텐츠를 놓을 수 없습니다. (GUIDES-44883)
- 테이블에 `scale` 특성을 적용할 때 테이블이 작성자 및 미리보기 모드에서 구성된 크기로 렌더링되지 않습니다. (GUIDES-45984)
- 페인트 또는 자르기 도구와 같은 외부 소스에서 복사한 이미지를 붙여넣으면 이미지가 주제에 삽입되지 않습니다. (GUIDES-45983)
- 키워드 맵에서 파생된 주제 제목에 사용된 `keyref`은(는) 브라우저를 새로 고친 후에도 저장한 후 목차 또는 주제 탭에 나타나지 않습니다. (GUIDES-45799)
- 주석이 있는 버전에 대해 나란히 보기를 사용하여 편집기에서 검토 작업을 열면 주제와 관련된 버전 대신 버전이 없음으로 표시됩니다. (GUIDES-45127)
- 검토 작업 페이지에 액세스할 때 주제 간의 페이지 나누기가 표시되지 않으므로 콘텐츠 섹션이 계속 렌더링됩니다. (GUIDES-46777)
- 검토 페이지 스타일이 새 편집기 경험과 일치하지 않아 일관되지 않은 시각적 경험이 발생합니다(GUIDES-46061).

## 자산 관리

- 맵에 DITA가 아닌 리소스(예: `.html`)를 가리키는 외부 `topicref`이(가) 포함된 경우 해당 미리 보기가 Assets UI에 표시되지 않습니다. (GUIDES-45409)
- 많은 소스 폴더를 선택할 때 [벌크 이동 도구] 대화 상자가 오버플로우되고 스크롤 막대가 없으므로 마우스 전용 사용자가 대상 경로 필드와 작업 버튼을 액세스할 수 없습니다. (GUIDES-45805)
- Assets UI에서 에셋을 복사하면 `DXML reprocessing failure`에 대한 알림을 받는 중입니다. 이 알림은 복사가 성공하여 오해의 소지가 있으며 받은 편지함 혼란을 초래합니다. (GUIDES-45012)
- 폴더 내의 Assets 왼쪽 레일에서 필터 패널을 열 경우 패널을 닫고 다시 열 때까지 검색 필드 및 패싯이 비활성화된 상태로 유지됩니다. (GUIDES-42652)
- 번역 워크플로 중에 생성된 콘텐츠 조각 및 중간 번역 에셋은 예약된 에셋 처리 작업에 의해 의도하지 않게 처리되어 로그에 예외가 발생하고 아직 준비되지 않은 에셋이 잘못 처리됩니다. (GUIDES-42582)

## 게시

- PDF 템플릿에서 `.plt` 및 `.css` 파일로 작업할 때 이러한 파일 형식에 적용할 수 없음에도 불구하고 마우스 오른쪽 단추 클릭 컨텍스트 메뉴에서 **ID 생성** 옵션을 사용할 수 있습니다. (GUIDES-45254)
- 브라우저를 새로 고친 후 새로 만든 네이티브 AEM Sites 사전 설정을 다시 열 때 변경 사항이 없는 경우에도 기본적으로 **저장** 단추가 잘못 활성화됩니다. (GUIDES-45171)
- 기본값으로 표시되는 네이티브 PDF 사전 설정을 복제할 때 복제도 기본 사전 설정으로 설정됩니다. 한 번에 하나의 사전 설정만 기본값으로 지정해야 합니다. (GUIDES-44786)
- `outputclass` 특성이 생성된 HTML 또는 기본 PDF 출력으로 전파되지 않으므로 MathML 방정식에 적용된 사용자 지정 클래스가 렌더링된 출력에 영향을 주지 않습니다. (GUIDES-44393)
- 새 AEM Sites 템플릿을 사용하여 생성된 AEM Sites 출력의 벌크 활성화에 실패하여 게시 인스턴스에 성공적으로 복제되지 못했습니다. (GUIDES-44049)
- DITA-OT 패키지에서 AEM Guides과 함께 번들로 제공되는 취약한 `jackson-databind` JAR(버전 2.9.8)가 식별됩니다. (GUIDES-43081)
- 맵 컬렉션에서 네이티브 AEM Sites 출력을 열면 리소스를 찾을 수 없다는 오류가 발생하여 생성된 출력에 액세스할 수 없습니다. (GUIDES-42065)
- 기본 PDF 생성 중에 DITA 맵의 `<reltable>` 요소가 무시되어 &quot;관련 개념&quot;, &quot;관련 작업&quot; 및 이와 유사하게 자동 생성된 상호 참조 섹션이 출력에서 누락됩니다. (GUIDES-38333)
- 기존 구성 요소 매핑을 사용하여 AEM Sites에 `processing-role=resource-only` 요소가 있는 DITA 맵을 게시하면 `topicgroup` 요소 및 특정 콘텐츠 구성과 같은 추가 시나리오에서 해당 요소에 대해 분리된 사이트 페이지가 생성됩니다. (GUIDES-37650)
- 이름이 긴 맵을 맵 컬렉션에 추가하면 맵 컬렉션 UI가 왜곡된 레이아웃으로 렌더링됩니다. 이 문제는 새 맵 컬렉션으로 해결되었습니다. (GUIDES-42062)
- 기본 PDF 엔진 v1을 사용하여 게시:
   - 특정 컨텐츠에 대한 기본 PDF 출력을 생성할 때, 여러 페이지에 걸쳐 전체 컨텐츠가 들어 있는 중간 HTML에도 불구하고 PDF에서 첫 번째 페이지만 렌더링됩니다. (GUIDES-28270)
   - 접근성 설정이 활성화된 기본 PDF 출력의 콘텐츠 읽기 순서가 잘못되었습니다. 바닥글의 페이지 번호는 끝이 아닌 기본 콘텐츠 앞에 읽혀집니다. (GUIDES-27790)
   - 기본 PDF 출력의 색상 막대는 전체 페이지 너비를 넘지 않고 페이지 크기를 사용자 지정할 때 겹쳐 일부 색상 상자가 숨겨집니다. (GUIDES-15505)
   - CSS `:is()` 의사(pseudo) 클래스 선택기가 기본 PDF 출력에서 사용되지 않으므로 브라우저 렌더링과 스타일이 다릅니다. (GUIDES-11328)

  >[!NOTE]
  >
  > 앞서 언급한 문제는 기본 PDF 엔진 v2에서 해결되었습니다. 자세한 내용은 [기본 PDF 엔진으로 작업 v2](../native-pdf/new-pdf-engine.md)을 참조하세요.

## 번역

- 에셋이 많은 맵에 기준선을 적용하면 선택한 언어에 대한 번역 보고서의 로드가 지연되며 이로 인해 보고서가 렌더링되기 전에 요청 시간 초과가 발생하는 경우가 있습니다. (GUIDES-45508)
- 선택한 언어 폴더에서 밑줄 형식(`da_dk`) 대신 하이픈이 연결된 로케일 코드(예: `da-DK`)를 사용하는 경우 Workspace 설정 아래의 번역 탭의 언어 그룹은 유지되지 않습니다. (GUIDES-37843)

## 검토

- **버전 기록** 아이콘의 도구 설명이 항목 이름 옆에 있는 검토 UI의 왼쪽 패널에 없습니다. (GUIDES-45511)
- 활성 검토 작업을 편집할 때 이미 검토에 포함된 항목을 제거한 다음 **업데이트**&#x200B;를 선택하지 않고 다시 추가하면 해당 항목에 대한 검토자 정보가 손실됩니다. (GUIDES-38774)
- 검토 UI의 콘텐츠 창에는 권장 1600px 해상도를 비롯한 특정 화면 해상도에 가로 스크롤 막대가 표시되므로 콘텐츠가 창 너비를 초과할 때 검토 주석이 숨겨집니다. (GUIDES-44082)

## 학습 콘텐츠

- 문제 은행에서 삽입 옵션을 사용하여 퀴즈에 질문을 추가할 때, 질문 ID가 유효함에도 불구하고 주관식 질문이 나열되지 않습니다. (GUIDES-44942)
- 문제 은행에서 삽입 옵션을 사용하여 질문을 삽입할 때 문제 은행에서 삽입 대화 상자가 열리면 예기치 않게 깜박이거나 크기가 조정됩니다. (GUIDES-45524)
- 문제 은행에서 삽입 옵션을 사용하여 질문을 삽입할 때 질문 제목에 이미지가 포함되어 있으면 오류가 발생합니다. (GUIDES-45383)
- SCORM 출력 템플릿을 선택하면 템플릿 제목을 수정할 때 애플리케이션 오류가 발생합니다. (GUIDES-45521)
- 하위 제목(`.vtt`) 파일이 폴더에 업로드된 후 저장소 또는 탐색기 보기에 표시되지 않습니다. (GUIDES-46014)
- HTML 미리 보기에서 비디오 콘텐츠를 로드할 때 원본 파일 경로를 유지하는 대신 원본(`src`) 경로에 추가된 렌디션 경로가 포함되어 비디오가 올바르게 렌더링되지 않습니다. (GUIDES-41776)
- Safari 브라우저에서 SCORM 출력을 게시하면 패키지가 zip 파일 대신 폴더로 다운로드되고 생성된 콘텐츠(예: 스트레치된 콘텐츠, 작동하지 않는 아코디언 등)에서 렌더링 및 기능 문제가 발생합니다. (GUIDES-45119)
- 과정에 대해 다음 버튼이 활성화되기 전에 지연이 관찰되어 학습자 탐색 경험에 영향을 줍니다. (GUIDES-45113)
- 미리 보기에 올바르게 표시되더라도 결과를 게시할 때 단답형 질문 스타일이 잘못 렌더링됩니다. (GUIDES-46478)
- 비디오가 포함된 강의에 대한 PDF 출력을 생성할 때 비디오 컨텐츠가 렌더링되지 않고 비디오 썸네일이나 포스터 이미지와 같은 자리 표시자 이미지 대신 파일 경로만 표시됩니다. 이 문제는 기본 PDF 출력 사전 설정에서 **멀티미디어 파일 포함** 옵션을 활성화하여 해결되었습니다(GUIDES-45117).
- CSS 의사 클래스 및 요소는 안내서의 편집기에서 흰색으로 렌더링되므로 보이지 않거나 편집기 배경에서 읽기 어렵습니다. (GUIDES-45116)

## 알려진 문제

Adobe은 2026.06.0 릴리스에 대해 다음과 같은 알려진 문제를 확인했습니다.

## 편집기 2.0

- Source과 작성자 모드 간을 전환하면 주제의 일부가 사라지거나 모드 간에 반영되지 않는 등 콘텐츠가 일치하지 않습니다. (GUIDES-47432)

- **변경 내용 추적**&#x200B;을 사용하여 개요 보기에서 작업할 때 변경 내용을 거부하면 수정된 특정 콘텐츠만 거부하는 대신 태그 내의 전체 콘텐츠가 제거됩니다. (GUIDES-48319)

- 폴더 프로필에서 `editor_toolbar.json`을(를) 사용하여 편집기 도구 모음을 사용자 지정할 때 미리 보기 모드의 **PDF으로 내보내기** 단추가 작업을 수행하지 않습니다. (GUIDES-47525)

- 삭제 작업을 수행하는 동안 이미지 맵, 구조화된 요소, 인라인 서식 태그 및 병합할 수 없는 블록 간에 커서 이동 및 탐색의 일부 사소한 불일치가 발생하여 예기치 않은 커서 또는 삭제 동작이 발생할 수 있습니다. (GUIDES-46756)

- 맵 편집기에서 끊어진 링크에 `Ctrl+click`을(를) 사용하면 응용 프로그램 오류가 트리거됩니다. (GUIDES-45544)

- 읽기 전용 콘텐츠(예: `conref` 단락) 바로 다음에 오는 단락 시작 부분에서 백스페이스를 누르면 편집 가능한 단락이 예기치 않게 삭제되거나 병합될 수 있으므로 편집 가능한 단락이 예기치 않게 삭제됩니다. (GUIDES-45049)

- 조건 표시기가 `bodydiv`과(와) 같은 요소에 적용되면 표시기가 전체 태그 보기의 인접 태그로 오버플로되어 잘못된 시각적 렌더링이 발생합니다. (GUIDES-44971)

- 편집기에서 작업할 때 `keyrefs`개 또는 맵에서 파생된 키(`term` 또는 `abbreviated-form` 요소에서 참조된 맵 내의 용어집 항목 포함)를 선택할 수 없으므로 작성 환경이 저하됩니다. (GUIDES-45790) <br> **해결 방법**: 특성 값을 사용하여 오른쪽 패널에서 `keyref`의 값을 변경할 수 있습니다.

- 개요 보기에서 항목을 닫았다가 다시 연 후 **텍스트 표시**&#x200B;를 다시 활성화하면 요소 태그와 함께 텍스트가 표시되지 않습니다. (GUIDES-48320) <br> **해결 방법**: **텍스트 표시** 옵션을 사용하도록 설정한 다음 주제를 다시 엽니다. 다시 열면 요소 태그와 함께 텍스트가 올바르게 표시됩니다.

- **요소 이름 바꾸기** 옵션을 사용하여 인라인 태그의 이름을 변경하면 이동 경로가 즉시 업데이트되지 않고 커서를 태그 안으로 이동하거나 보기 모드를 변경한 후에만 변경 내용을 반영합니다. (GUIDES-44993) <br> **해결 방법**: 이동 경로를 업데이트하기 위해 인라인 태그의 이름을 바꾼 후 브라우저를 새로 고치십시오.

- MathML 방정식이 `conref`(으)로 삽입되면 올바르게 렌더링되지 않습니다. (GUIDES-46601) <br> **해결 방법**: `<p>` 요소 내에서 MathML 식을 래핑하고 해당 `<p>` 요소를 conref 소스로 사용합니다.

## 검토

- 프로젝트 팀 외부의 사용자에게 검토 작업이 재할당되면 사용자는 프로젝트 구성원이 없어도 검토 작업을 수행할 수 있지만 검토자 가시성, 검토 상태 추적 및 위임 알림이 제대로 작동하지 않습니다. (GUIDES-46602)

## 게시

- 조건부 프로파일링을 통해 모든 글머리 기호 목록 항목을 제외하는 DITAVAL 필터링으로 컨텐츠를 게시할 때 출력에서 전체 목록 구조를 제거하는 대신 빈 글머리 기호 마커가 잘못 유지됩니다. (GUIDES-47238)

- 새 기준선이 있는 기본 AEM 사이트 출력을 게시할 때 주제 목록이 비어 있고 선택한 기준선에 포함된 주제를 표시할 수 없습니다. (GUIDES-46480) <br> **해결 방법**: 구성 관리자를 통해 구성할 수 있는 이전 기준선을 사용하여 기본 AEM 사이트 출력을 게시합니다.
