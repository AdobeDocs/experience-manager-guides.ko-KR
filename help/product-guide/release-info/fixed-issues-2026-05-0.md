# 2026.05.0 릴리스의 문제가 해결되었습니다.

이 문서에서는 Adobe Experience Manager Guides as a Cloud Service의 2026.05.0 릴리스의 다양한 영역에서 수정된 버그를 다룹니다.

새로운 기능 및 향상된 기능에 대한 자세한 내용은 [2026.05.0 릴리스의 새로운 기능](whats-new-2026-05-0.md)을 참조하세요.

[2026.05.0 릴리스의 업그레이드 지침](upgrade-instructions-2026-05-0.md)에 대해 알아봅니다.

## 작성

- **파일 선택** 대화 상자를 사용하여 편집기에서 이미지를 선택하면 래스터 형식(예: JPG, PNG, GIF)만 표시됩니다. 벡터 파일(.ai 및 .eps 등)은 표시되지 않으며 선택할 수 없습니다. (GUIDES-45110)
- 업그레이드 시 `ui_config.json`의 기본값이 `true`(으)로 설정되어 있더라도 기본적으로 `tagsView` 설정이 꺼집니다. (GUIDES-44651)
- 편집기에서 파일 참조는 `xmleditor.uuid` 구성에도 불구하고 파일 경로 대신 GUID로 표시됩니다. (GUIDES-42438)
- 키 값이 유사한 제목 구성표가 DITA 주제에 적용되면 거의 동일한 색상으로 강조 표시되므로 해당 구성표를 구별하고 적용되는 구성표를 식별하기 어렵습니다. (GUIDES-38472)
- 작성자 보기에서 제목 구성표 맵을 편집할 때 `hasInstance` 요소를 추가하면 파일 선택 대화 상자가 자동으로 트리거되어 작성자가 AEM 에셋을 가리키는 원치 않는 `href`을(를) 삽입해야 합니다. 또한 이러한 맵에 대해 **콘텐츠 속성** 패널을 로드하지 못하므로 작성자가 작성자 보기에서 요소 특성을 업데이트할 수 없으며 특성을 업데이트하려면 Source 보기를 사용해야 합니다. (GUIDES-38164)
- `.ditaval` 파일을 편집할 때 작성자 보기로 전환한 다음 Source 보기로 돌아가면 Source 보기에 추가된 모든 XML 주석이 제거됩니다. (GUIDES-33228)


## 자산 관리

- 이름에 공백이 있는 폴더에 주제를 만들면 하이픈으로 공백을 대체하는 중복 폴더가 잘못 생성되고 주제가 원래 폴더 대신 해당 폴더에 저장됩니다. (GUIDES-41938)
- 대용량 맵을 처음 변환하는 동안 대상 언어에 대해 빈 XML 파일이 생성되므로 서버 로드가 증가하고 성능이 느려집니다. (GUIDES-41613)
- DB 기반 환경에서는 올바른 내부 DITA 링크가 편집기 및 게시된 출력에서 올바르게 작동하더라도 **자산 속성**&#x200B;에서 끊어진 링크로 표시됩니다. (GUIDES-35048)

## 게시

- 폴더 프로필의 출력 사전 설정 변경 사항이 기존 맵에 적용되면 AEM Sites 사전 설정에 대해 저장된 **게시 컨텍스트**&#x200B;가 재설정됩니다. (GUIDES-38377)
- 맵 또는 북맵의 제목 내에서 `tm` 요소를 사용할 때 기본 PDF 출력의 표지 페이지에 상표 기호(®)가 렌더링되지 않습니다. (GUIDES-28832)
- 기본 PDF 템플릿을 사용하여 맵을 게시할 때 **맵 제목**&#x200B;에 맵 `title`에서 사용되는 하위 요소의 콘텐츠가 포함되어 있지 않으며 해당 콘텐츠 필터링이 제목에 적용되지 않습니다.(GUIDES-33730)
- `chunk="to-content"`을(를) 사용하는 `topicref`을(를) 가리키는 경우 AEM Sites 출력의 교차 맵 피어 링크를 확인하지 못했습니다. (GUIDES-37873)
- 게시 중에 DITAVAL 기반 플래그 지정과 관련된 파일은 출력에서 예상되는 상대 위치에 남아 있지 않고 새 시스템 생성 폴더로 이동됩니다. (GUIDES-37564)
- 조건이 충돌하는 여러 DITAVAL 파일을 사용하면 기본 PDF 출력이 실패합니다. (GUIDES-37484)

## 베이스라인

- 사용자 지정 DITA-OT를 사용할 때 맵 내의 주제 참조가 직접 참조되더라도 간접적으로 잘못 표시됩니다. 이 문제는 새 기준선 경험으로 해결되었습니다. (GUIDES-19286)
- `scope="peer"`이(가) 있는 참조가 기준 컨텍스트에 잘못 포함되어 게시가 예상보다 오래 걸립니다. 이 문제는 새 기준선 경험으로 해결되었습니다. (GUIDES-30048)

## Platform

- 큰 주제나 맵을 열면 작성자 인스턴스가 응답하지 않고 경우에 따라 다시 시작해야 합니다. (GUIDES-43547)

## 편집기 2.0에서 사용할 수 있는 해결된 문제

편집기 2.0(즉, 새 편집기)을 사용할 때 다음 문제가 해결되었으며 더 이상 발생하지 않습니다.

- 테이블에서 둘 이상의 열을 삭제하면 테이블 구조가 일관되지 않거나 손상됩니다. (GUIDES-35438)
- 병합된 셀이 들어 있는 테이블에서 열을 삭제하면 빈 열이 새로 추가됩니다. (GUIDES-30147)
- 탐색 표시 메뉴에서 기존 표에 새 행을 삽입하면 표 구조가 예기치 않게 변경되어 테두리가 누락되고 추가 열이 발생합니다. (GUIDES-29194)
- 작성자 보기에서 표 행을 복사하여 붙여넣으면 표 내에서 콘텐츠를 새 행으로 삽입하는 대신 표 외부에 배치됩니다. (GUIDES-24372)
- 작성자 모드에서 마우스 끌기를 사용하여 선택한 섹션 요소를 복사하여 붙여넣으면 섹션 구조를 유지하는 대신 단락 `(<p>)` 요소로 변환됩니다. (GUIDES-30023)
- step 또는 uicontrol과 같은 요소 내에 강조 표시된 텍스트를 편집하면 선택한 텍스트가 올바르게 대체되지 않고 대신 추가 또는 앞에 추가되어 유효성 검사 오류가 발생합니다. (GUIDES-24371)
- 상대 값을 사용하여 표 열 너비를 설정하면 나머지 열이 비례적으로 조정되지 않으므로 표 레이아웃이 잘못 정렬됩니다. (GUIDES-26109)
- 복사된 주제 제목을 태그가 비활성화된 상태로 붙여넣으면, 첫 번째 붙여넣기에 잘못된 스타일이 적용되고 제목 대신 콘텐츠 속성의 유형이 주제로 할당됩니다. (GUIDES-28838)
- 콘텐츠의 큰 부분을 편집할 때 의도하지 않은 스크롤 이동으로 인해 편집기 보기가 활성 콘텐츠에서 벗어납니다. (GUIDES-35436)
- 요소에서 백스페이스를 사용하면 편집기는 커서 위치에 관계없이 주제의 맨 위로 스크롤됩니다. (GUIDES-32520)
- 왼쪽 또는 오른쪽 화살표 키를 사용하여 인라인 태그 밖으로 이동하면 커서가 첫 번째 시도에서 예기치 않게 이동합니다. (GUIDES-26363)
- AEM Spellcheck는 기본 en-US 언어에서만 작동하며 다른 로케일은 사용하지 않습니다. (GUIDES-14731)
- 편집기에서 큰 DITA 주제의 잠금이 해제된 경우 동일한 주제가 중복 탭에서 다시 열립니다. 또한 실제 태그 수 대신 `NaN`이(가) 표시되는 태그 제한 경고가 트리거됩니다. (GUIDES-34008)
- 읽기 전용 또는 잠긴 주제에 대해 Acrolinx 제안이 편집기에서 올바르게 강조 표시되지 않습니다. (GUIDES-29614)
- 작성자 보기에서 머리글 행 없이 새 `reltable`을(를) 만들 때 첫 번째 셀에 주제가 추가된 후 표 레이아웃이 변경되어 다음 열이 축소되고 관련 주제를 배치하기가 어렵습니다. (GUIDES-19555)
- 작성자 모드에서 `xref` 링크를 작은 테이블 셀에 추가하면 해당 링크는 셀 내에 포함되지 않고 동일한 행의 인접한 셀에 나타납니다. (GUIDES-5489)
- 작성자에서 Source 보기로 전환하는 경우 커서 위치가 유지되지 않고 편집기가 맨 위로 다시 스크롤됩니다. 또한 긴 주제에서는 커서 위치가 손실되어 작성자 보기와 Source 보기 사이를 전환할 때 중간이나 맨 위로 임의로 이동합니다. (GUIDES-18114, GUIDES-21164)
- `<li>` 요소 내에서 *Enter*&#x200B;을(를) 누르면 새 `<li>`이(가) 만들어지지만 이전 `<li>`(으)로 다시 이동하고 *Enter*&#x200B;을(를) 누르면 현재 항목의 콘텐츠가 `<p>` 요소로 잘못 변환되어 목록 구조가 손상됩니다. (GUIDES-27505)

## 알려진 문제

Adobe은 2026.05.0 릴리스에 대해 다음과 같은 알려진 문제를 확인했습니다.

- 맵에 DITA가 아닌 리소스(예: `.html` 또는 `.htm`)를 가리키는 외부 `topicref`이(가) 포함된 경우 해당 미리 보기가 Assets UI에 표시되지 않습니다. (GUIDES-45409)
- 편집기 미리 보기에서 올바르게 렌더링되지만 Assets UI에서 `conref`을(를) 사용하여 참조된 콘텐츠가 DITA 주제에 대해 표시되지 않습니다. (GUIDES-45505)<br>**해결 방법**: 이러한 콘텐츠의 경우 편집기의 미리 보기를 사용할 수 있습니다.
- `xmleditor.uniquefilenames` 속성이 활성화된 경우 템플릿을 사용하여 만든 새 주제에는 주제 제목이 포함되지 않습니다. (GUIDES-44737)
- `getAsset`, `startAssetProcessing` 및 `getAssetProcessingStatus` API는 Java SDK을 통해 사용할 수 없습니다.
