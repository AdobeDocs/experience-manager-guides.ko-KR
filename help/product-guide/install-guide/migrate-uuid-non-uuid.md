# 비 UUID에서 UUID로의 콘텐츠 마이그레이션 {#id226TI0U20XA}


사용 중인 Experience Manager Guides의 현재 버전을 기반으로 하여 UUID가 아닌 콘텐츠를 UUID로 마이그레이션할 수 있습니다.

>[!IMPORTANT]
>
> UUID 서버로 콘텐츠를 마이그레이션하기 전에 호환되는 AEM Guides 버전이 있는 비 UUID 서버가 설치되어 있는지 확인하십시오.

## 호환성 매트릭스

다음 매트릭스를 사용하여 UUID가 아닌 현재 버전을 기반으로 올바른 마이그레이션 경로를 파악하십시오. 이렇게 하면 마이그레이션 후 원활하게 전환할 수 있습니다.

| 마이그레이션에 필요한 UUID가 아닌 버전 | 마이그레이션 후 UUID 버전 | 마이그레이션 후 지원되는 업그레이드 경로 |
|---|---|---|
| 4.3.1 비 UUID | 4.3.2 UUID | 버전 4.3.2 UUID로 마이그레이션한 후 4.6.0 (UUID)을 직접 설치해야 합니다. 4.6.0을 사용하고 나면 버전 5.1.0으로 업그레이드한 다음 5.1.0 서비스 팩 3을 설치하십시오. |
| 4.6.0 서비스 팩 4 비 UUID | 4.6.1 UUID | 버전 4.6.1 UUID로 마이그레이션한 후 직접 5.1.0 (UUID)으로 업그레이드해야 합니다. 업그레이드가 완료되면 버전 5.1.0 서비스 팩 3을 설치합니다. |

## 마이그레이션 시간 예상

마이그레이션 유틸리티는 에셋당 평균 ~50ms 속도로 에셋을 처리합니다. 다음 표에서는 64개의 vCPU, 128GB RAM 및 SSD 백업 스토리지로 구성된 시스템의 마이그레이션 시간 예상을 제공합니다. 렌디션이나 고해상도 바이너리가 많은 더 큰 저장소나 에셋의 경우 메모리 요구 사항이 증가할 수 있습니다.

>[!NOTE]
>
> 실제 마이그레이션 시간은 하드웨어 성능, 스토리지 처리량, 동시 AEM 작업 및 전체 시스템 로드에 따라 달라질 수 있습니다.
