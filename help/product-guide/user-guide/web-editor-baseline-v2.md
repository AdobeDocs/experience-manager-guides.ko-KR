# Experience Manager Guides의 새 기준선(Beta)

>[!NOTE]
>
> 이 문서는 Experience Manager Guides 2026.03.0 릴리스에서 사용할 수 있는 향상된 성능과 안정성을 제공하는 현재 *Beta* 기능으로 사용 가능한 새 기준선 에도 적용됩니다. 설정에서 새 기준선 기능을 활성화하려면 고객 성공 팀에 문의하십시오.

새로운 기준선 기능은 크고 복잡한 맵과 관련된 중요한 안정성 및 성능 문제를 해결합니다. 이 제품에는 보다 빠르고 안정적이며 일관된 기본 경험을 제공하는 다시 설계된 기본 아키텍처가 포함되어 있습니다. 자세한 내용을 살펴보기 전에 새로운 기준선 기능의 기능을 강조 표시하는 짧은 연습 비디오입니다.

>[!VIDEO](https://video.tv.adobe.com/v/3483154/aem-guides)

새로운 기준선 모델은 일반적인 불만 사항을 해결하여 기준선 처리를 강화합니다.

- 대형 기준선을 사용하여 작업할 때 로드 속도가 느리고 응답성이 떨어짐
- 부분 업데이트 또는 유효성 검사 실패로 인해 일관성 없는 기준선 상태
- 광범위한 기준 컨텐츠 관리 시 제한된 가시성 및 제어
- 기준 생성, 업데이트 또는 재구축 중 성능 병목 현상

다음 섹션에서는 향상된 기능, 마이그레이션 전에 고려해야 할 주요 동작 변경 사항, 새 기준 요소로의 마이그레이션 및 사용 지침을 포함하여 새 기준 요소 모델에 대해 설명합니다.

- [새 기준선에 도입된 주요 개선 사항](#key-enhancements-introduced-in-the-new-baseline)
- [새 기준선으로 마이그레이션하기 전에 알아야 할 동작 변경](#behavior-changes-to-know-before-migrating-to-the-new-baseline)
- [새 기준선으로 마이그레이션](#migrate-to-new-baseline)
- [새 기준선 사용](#use-the-new-baseline)

## 새 기준선에 도입된 주요 개선 사항

새로운 기준선은 작업 방식을 변경하지 않고도 기준 요소 관리를 더 빠르고 쉽게 확장할 수 있도록 대폭 향상된 기능을 제공합니다. 다음에 대한 새 기준선으로 이동하는 것이 좋습니다.

- **향상된 성능 및 확장성:** 기본 데이터 모델 및 렌더링 동작은 증분 로드 및 간소화된 데이터 구조를 사용하여 응답성을 향상하고 큰 기준선을 사용하여 효율적으로 확장할 수 있도록 최적화되었습니다.
- **강력한 UI 및 백엔드 일관성:** 기준선에 대한 모든 변경 사항(예: 버전 또는 종속성 업데이트)은 이제 성공적인 백엔드 유효성 검사 후에만 UI에 반영되므로 잘못된 기준선을 만들 수 없습니다.
- **필터링, 정렬 및 탐색:** 기준선은 문서 상태, 레이블, 파일 형식, 참조 형식 및 전체 기준선에서 GUID 기반 검색을 포함하여 여러 특성에 대한 포괄적인 필터링을 지원합니다. 페이지 매김은 레이블이 없는 파일을 포함하는 옵션과 함께 큰 기준선에 대해 지원됩니다.
- **종속성 영향에 대한 명확한 가시성:** 종속성 영향(추가 또는 제거된 종속성에 대해)은 버전 변경 사항이 적용되기 전에 미리 보기로 표시되므로 변경 사항을 적용하기 전에 검토할 수 있습니다.
- **보다 유연한 레이블 관리:** 기준선 내의 버전 간에 레이블을 이동할 수 있으므로 다양한 주제 버전에서 레이블을 관리할 때 보다 유연합니다.
- **결정적 편집 및 저장 동작:** 기준선 편집은 행 수준 업데이트를 지원하고, 리소스 사용량이 많은 데이터(버전 트리 및 종속성 차이 등)를 버전 업데이트 중에만 로드하며, 저장 작업을 한 단계에서 결정적으로 완료함으로써 예기치 않은 저장 실패와 부분 업데이트를 줄일 수 있습니다.
- **보다 안정적인 기준 요소 만들기:** 기준 요소는 런타임 구문 분석이 아닌 저장된 참조 데이터를 사용하여 만듭니다. 불완전하거나 잘못된 기준 요소를 방지하기 위해 필요한 버전 정보가 미리 확인됩니다.
- **API 및 자동화 지원:** 새 기본 모델은 REST API 및 Java SDK을 통해 완전히 지원되므로 자동화와 외부 워크플로우와의 통합을 가능하게 합니다.


## 새 기준선으로 마이그레이션하기 전에 알아야 할 동작 변경

새 베이스라인 모델로 마이그레이션하기 전에 다음 비헤이비어 변경 사항을 검토하십시오. 이러한 변경 사항은 기준선을 만들고, 업데이트하고, 관리하는 방법에 영향을 주며 기존 워크플로우에 영향을 줄 수 있습니다.

| 영역 | 변경(설명) |
|------|-------------|
| **참조 확인** | 다이렉트 맵 참조는 **DIRECT**(으)로 분류됩니다. 잘못된 참조를 건너뛰었으며 `reltable`의 참조는 계속 제외됩니다. |
| **자동으로 선택** | 직접 참조를 확인하기 전에 버전 선택을 즉시 평가하여 정확한 버전 해결을 보장합니다. |
| **기준 생성 규칙** | 버전 **1.0**&#x200B;은(는) 필수입니다. 누락되거나 모호한 버전이 있는 기준선은 마이그레이션 후 다르게 해결될 수 있습니다. |
| **마이그레이션 처리** | 잘못된 참조를 건너뜁니다. **DIRECT** 참조가 우선하며, 고정되지 않은 참조가 최신 버전으로 이동하고, 추가 메타데이터가 버전 **5.0**&#x200B;에서 계속 추가됩니다. |
| **기준 데이터 모델** | 새로운 그래프 기반 기준선 모델은 변경 가능한 필드를 제거하며 이전 기준선 모델과 역으로 호환되지 않습니다. |
| **API 사용** | 기준 작업은 REST API 및 Java SDK을 통해 지원됩니다. 원시 베이스라인 객체가 더 이상 노출되지 않습니다. |
| **버전 제거** | 마이그레이션 후 버전 삭제는 새 기준 저장소에 저장된 기준선만 고려합니다. |

## 새 기준선으로 마이그레이션

고객 성공 팀에서 기능을 활성화한 후에는 기존 기준선을 새 기준선으로 마이그레이션해야 합니다.

기존 베이스라인을 새 베이스라인으로 이전하려면 다음 단계를 수행합니다.

1. 맨 위에 있는 Adobe Experience Manager 로고를 선택하고 **도구**&#x200B;를 선택합니다.
1. **도구** 패널에서 **안내서**&#x200B;를 선택합니다.
1. **일괄 프로세서** 타일을 선택하십시오.

   ![흐름 자산 프로세서](images/flow-asset-processor.png){align="left"}

   **Guides 대량 프로세서** 페이지가 표시됩니다.

1. 페이지 오른쪽 상단에서 **새 프로세스**&#x200B;를 선택하여 새 처리 작업을 시작합니다.

   **새 프로세스** 대화 상자가 표시됩니다.

1. 대화 상자에서 다음 세부 사항을 입력합니다.

   1. **기능 유형**: 드롭다운에서 **기준선**&#x200B;을 선택합니다.
   1. **폴더 및 파일 선택**: 처리할 하나 이상의 폴더 및 파일을 탐색하고 선택합니다.
   1. **무시할 폴더 선택**: 선택적으로 선택한 상위 폴더 내에서 마이그레이션에서 제외할 하위 폴더를 선택하십시오.

   ![new-process-baseline](images/new-process-baseline.png){align="left"}

1. **만들기**&#x200B;를 선택합니다.

**자산 처리가 트리거됨**&#x200B;을 보여주는 팝업이 표시됩니다. 페이지에서 처리 작업의 상태를 볼 수 있습니다.

**로그 보기**&#x200B;를 선택하여 마이그레이션 작업에 대한 로그를 확인하고 다운로드할 수도 있습니다.

![view-logs-baseline](images/view-logs-baseline.png){align="left"}

로그 보고서는 마이그레이션된 맵 수, 마이그레이션된 기준선 및 관련 세부 정보를 포함하여 마이그레이션에 대한 세부 정보를 제공합니다.

![logs-detail-baseline](images/logs-detail-baseline.png){align="left"}

>[!NOTE]
>
> 마이그레이션 도중, 특히 작업 중인 복사본에서 기준선 편집을 수행하지 않아도 오류를 방지할 수 있습니다. 마이그레이션 후, 일부 기준선은 버전이 누락된 경우 다시 작성해야 할 수 있습니다.

## 새 기준선 사용

새 기준선 모델은 Experience Manager Guides의 기존 기준선 기능과 동일한 워크플로 및 사용자 인터페이스를 사용합니다. 사용 가능한 옵션을 사용하여 [맵 콘솔에서 기준선을 만들고 관리](./web-editor-baseline.md)할 수 있습니다.

>[!NOTE]
>
> 새 베이스라인 모델은 맵 대시보드에서 베이스라인을 만들고 관리하는 것을 지원하지 않습니다.

이 섹션에서는 새 베이스라인 모델에 도입된 변경 사항 및 개선 사항만 설명합니다. 일반적인 기준 워크플로우는 명시적으로 언급되지 않는 한 변경되지 않습니다.

**새 기준 UI에서 사용 가능한 새로운/향상된 옵션**

**새 기준선 모델**&#x200B;을 사용하여 만든 기준선을 사용하여 작업할 때 다음 업데이트가 적용됩니다.

- 옵션 메뉴의 **기준선 내보내기** 옵션 이름이 수동 및 자동 업데이트를 모두 사용하여 만든 기준선의 **다운로드**(으)로 바뀝니다.

  ![](images/baseline-v2-options-menu.png)

- 동적 기준선은 **기준선** 패널에서 직접 열 수 있으며 [옵션] 메뉴에서 사용 가능한 작업을 사용하여 관리할 수 있습니다.

  ![](images/baseline-v2-dynamic-baseline-new.png)

  새 베이스라인 모델을 사용하여 생성된 동적 베이스라인에 대해 도입된 새로운 옵션을 사용할 수도 있습니다.
   - **속성 편집**: 기존 기준선의 속성을 편집할 수 있습니다.
   - **다시 빌드**: 변경 사항이 발생할 때마다 동적 기준선을 다시 빌드할 수 있습니다.

     ![다시 빌드-기준선](images/rebuild-baseline.png){align="left"}

- **다운로드** 작업은 페이지 매김된 다운로드를 지원합니다. 적용된 필터와 일치하는 모든 기준선 콘텐츠는 현재 페이지에 표시되는 콘텐츠뿐만 아니라 다운로드에 포함됩니다.
- 파일 이름 또는 파일 위치 외에 GUID로 파일을 필터링합니다. **레이블이 없는 파일을 필터링**&#x200B;하는 추가 옵션도 사용할 수 있습니다.

  ![](images/baseline-v2-filters.png)
- 새 베이스라인 모델은 결정론적 편집을 지원하므로 검증된 종속성 해결로 한 번에 하나의 참조를 업데이트할 수 있습니다.

  +++새 베이스라인의 참조를 편집하는 단계

  베이스라인을 편집하려면 다음 단계를 수행하십시오.

   - **기준선** 패널에서 기준선을 엽니다.

     베이스라인의 참조를 테이블 형식으로 볼 수 있습니다.

   - 편집할 파일로 이동하여 그 위에 놓습니다.
   - **편집** 아이콘을 선택합니다.

     ![edit-baseline-icon](images/edit-baseline-icon.png){align="left"}

     **버전 편집** 대화 상자가 표시됩니다.
   - **버전** 드롭다운에서 필요한 버전을 선택합니다(예: 버전 1.0에서 1.1로 변경).


     ![edit-version-baseline](images/edit-version-baseline.png){align="left"}

     추가되고 제거된 종속성은 평가되어 미리보기로 표시됩니다. 변경 사항을 적용하기 전에 검토하십시오.

     ![](images/baseline-v2-version-added.png)

     종속성 변경이 감지되지 않으면 빈 상태 메시지가 표시됩니다.

   - 변경 내용을 적용하려면 **업데이트**&#x200B;를 선택하세요.

  베이스라인이 선택한 버전으로 업데이트됩니다.
